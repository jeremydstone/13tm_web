# Working in this repo

Static website for the band **13 Til Midnight** — **live at https://13tilmidnight.com** via
GitHub Pages. Plain HTML/CSS/JS, **no build step, no framework, no dependencies in the shipped
site** (the only `node_modules` are dev tooling). This is a **public** repo.

## Before you change anything
- **Always run `npm test` and get it green before opening a PR.** It runs html-validate,
  stylelint, eslint, and custom validators for links, `data/shows.json`, and
  `assets/photos/photos.json` (plus a self-test of the shows validator).
- **`main` is branch-protected:** no direct pushes; every change merges via a PR that must pass
  the **"Lint & verify"** CI check. Branch fresh from `main`. Pushing to `main` auto-deploys to
  the live site in ~1 min — don't break it.
- This project's owner drives merges with the `/makepr` and `/mergepr` skills — don't open PRs
  or merge unprompted.

## Things that will bite you if you don't know them
- **Clean URLs.** Pages live in folders as `index.html` (`/pics/`, `/videos/`, `/contact/`,
  `/epk/`; root is `index.html`). Because of this, **all asset/nav links are root-absolute**
  (`/css/styles.css`, `/assets/…`, `/pics/`) — never relative. Keep it that way; relative paths
  break from a subfolder page. This also means it must be served from the domain root (it is).
  Test locally with `python3 -m http.server` (not `file://`).
- **Content is data-driven, not hand-written HTML:**
  - Shows → `data/shows.json` (rendered by `js/shows.js`, which splits upcoming/past by today's
    date and groups past by year). `scripts/check-shows.mjs` validates it **strictly** (real
    `YYYY-MM-DD` dates, required date+title, button must be `{url,text}` with a valid http(s)
    url, and **no unexpected fields**). A non-technical editor updates this via the GitHub web
    UI — see `SITEUPDATE.md`. Don't loosen that validation.
  - Photos → `assets/photos/full/` + `assets/photos/photos.json`; thumbnails are generated by
    `npm run thumbs` (macOS `sips`). The gallery (`js/gallery.js`) is masonry + lightbox and
    takes an optional `data-limit` (the EPK reuses it for a 12-photo preview).
- **Design tokens** live as CSS custom properties in `css/styles.css` (`--accent`,
  `--accent-grad`, `--btn-grad`, `--font-display`/`--font-body` — both Inter). Prefer editing
  these over hardcoding colors/fonts.
- **No hover effects on the show cards** and the SHOWS/Videos pages render immediately (no
  scroll-reveal) — both deliberate fixes for scroll-jank; don't reintroduce hover/transform on
  elements the cursor scrolls over.
- **Keep PII out of this public repo** (the owner removed a contact email for this reason).

See `README.md` for full details and `SITEUPDATE.md` for the non-technical show-editing guide.
